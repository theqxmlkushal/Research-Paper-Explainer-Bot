## 📌 Project Overview

A fully usable, production-style application that lets you **chat with any research paper**.
Upload a PDF, and the bot answers your questions using a hand-built RAG (Retrieval-Augmented Generation) pipeline — no LangChain, no LlamaIndex, just plain Python so every step is transparent and educational.

### Key features
| Feature | Description |
|---|---|
| 💬 **Chat interface** | Ask questions in a clean chat UI with bot/user bubbles |
| 📝 **Auto-summariser** | 5-bullet-point summary generated by the LLM |
| 🔍 **RAG Explorer** | Visualise which chunks were retrieved and their similarity scores |
| 📐 **How RAG Works tab** | Math equations + code for every pipeline step |
| 📜 **History + Export** | Download your session as TXT or JSON |
| 🎛️ **Tunable settings** | Chunk size, overlap, top-K, temperature, model — all in the sidebar |

## 🛠️ Technologies / Concepts Used

| Technology | Role |
|---|---|
| **Streamlit** | Frontend + backend (single-file app) |
| **PyPDF2** | Extract raw text from uploaded PDFs |
| **SentenceTransformers** (`all-MiniLM-L6-v2`) | Convert text chunks → dense vectors (embeddings) |
| **FAISS** (`IndexFlatL2`) | Fast nearest-neighbour search over embeddings |
| **Groq API** (Llama 3 / Mixtral) | LLM for answer generation |
| **Plotly** | Interactive bar charts for similarity scores |
| **RAG Pipeline** | Retrieve relevant chunks before generating answers |
| **Sliding-window chunking** | Overlapping splits to preserve context at boundaries |
| **L2 distance → similarity** | `sim = 1 / (1 + L2_dist)` to score relevance |

## 🔁 RAG Pipeline (Step by Step)

```
PDF Upload
   │
   ▼
PDF Parsing (PyPDF2)          → raw text string
   │
   ▼
Sliding-Window Chunking       → list of overlapping text chunks
   │
   ▼
SentenceBERT Embeddings       → matrix of shape [n_chunks × 384]
   │
   ▼
FAISS IndexFlatL2             → searchable vector index
   │            │
   │            ▼  (at query time)
   │       embed(query) → query vector
   │            │
   │       nearest-neighbour search
   │            │
   │       top-K chunks (most similar)
   │            │
   ▼            ▼
Groq LLM  ← context + query prompt
   │
   ▼
Answer ✅
```

### Math behind retrieval

```
Embedding:    E(text) ∈ ℝ³⁸⁴              (MiniLM)

L2 distance:  dist(q, c) = √ Σᵢ (qᵢ − cᵢ)²

Similarity:   sim(q, c)  = 1 / (1 + dist(q, c))    ∈ (0, 1]

Top-K:        argmin_K  dist(q, cᵢ)   for all chunks cᵢ
```

## 💡 Key Design Decisions

1. **No framework (LangChain / LlamaIndex)** — Everything is hand-rolled so students can see exactly what is happening inside popular frameworks.
2. **Streamlit only** — Single-file keeps deployment trivial; `streamlit run streamlit_app.py` is all you need.
3. **FAISS `IndexFlatL2`** — Exact search is fine for paper-sized corpora (< 5 000 chunks). For larger scale, switch to `IndexIVFFlat`.
4. **Groq over OpenAI** — Free tier, extremely fast inference (400+ tokens/sec), OpenAI-compatible API.
5. **Overlapping chunks** — 50-char overlap stops answers from being cut across chunk boundaries.

## ⚙️ Configurable Parameters

| Parameter | Default | Effect |
|---|---|---|
| `chunk_size` | 500 chars | Larger = more context per chunk |
| `chunk_overlap` | 50 chars | Overlap to avoid losing context at edges |
| `top_k` | 3 | Number of chunks sent to LLM |
| `temperature` | 0.7 | Creativity of answers (0 = deterministic) |
| `max_tokens` | 600 | Maximum answer length |
| `embedding_model` | MiniLM-L6-v2 | Swap for `all-mpnet-base-v2` for better accuracy |
